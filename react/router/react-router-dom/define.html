定义路由 ​

路由 ​

前端路由：根据不同的 url 地址返回对应的页面效果

服务端路由：根据不同的 请求（url + method）-返回对应的数据或文件

配置型路由 ​

jsx
// createBrowserRouter：使用 DOM 历史记录 history API 来更新 URL 并管理历史记录堆栈。
import { createBrowserRouter, RouterProvider } from "react-router-dom";

// 路由器对象
const router = createBrowserRouter([])

function App() {
    return <RouterProvider router={router}></RouterProvider>
};

TIP

根路由对象定义的组件在被访问时会使用 RouterProvider 组件作为渲染出口。

声明式路由 ​

所有配置效果全部由组件方式实现

jsx
import { BrowserRouter, HashRouter, Routes, Route, Navigate } from "react-router-dom";
// BrowserRouter, HashRouter 路由器组件
// 需要作为项目的根组件
// 对内部嵌套的子组件进行管理，根据浏览器地址决定展示的内容
// Routes 路由组件的出口，Routes内部只会产出一个路由组件 效果类似于 Vue 中的 router-view
// Route 路由组件，作为路由器组件的子组件，定义访问路径和展示组件的映射关系
// Navigate：导航组件，是一个行为组件，当使用组件时，会在页面内产生跳转效果

import Index from "../views/index";
import Nest from "../views/nest";

import Sports from "../components/news/sports";
import Money from "../components/news/money";
import Funny from "../components/news/funny";

const App = () => {
  return (
    // 路由模式
    <HashRouter>
      {/* 根组件路由出口，一般情况下，一个项目使用一个就可以了 */}
      <Routes>
        {/* 根据路径展示组件：路由配置对象的组件形式，所有配置项都使用 props 形式进行定义
        路由组件的匹配方式是从上到下依次匹配的，如果找到一个，后续均不执行
        通用匹配组件都需要放在最后 */}
        <Route path="/" element={<Index />}></Route>
        <Route path="nest" Component={Nest}>
          {/* 定义嵌套路由的组件，访问嵌套组件的路径是 依次将 path 属性的值进行拼接 */}
          {/* path="" 等效于 index 属性 */}
          {/* <Route path="" element={<Sports />}></Route> */}
          {/* <Route index element={<Sports />}></Route> */}

          <Route index element={<Navigate to={"sports"}></Navigate>}></Route>
          <Route path="sports" element={<Sports />}></Route>
          <Route path="money" element={<Money />}></Route>
          <Route path="funny" element={<Funny />}></Route>
          <Route path="*" element={<div>Nest：404</div>}></Route>
        </Route>
        {/* * 表示路径匹配模式，能够匹配任意路径，一般用于处理 404 页面 */}
        <Route path="/*" element={<div>404</div>}></Route>
      </Routes>
    </HashRouter>
  );
};

export default App;
